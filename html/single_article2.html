    Why do I need Javascript data validation

    The one of the solutions to the problem of server side overloading is data validation. Data validation ensures that your data is complete, unique, and the range of values is consistent with what you expect before being processed.The best approach for data validation is to use both client-side and server-side validation.

    Most of the people prefer to use one methods of data validation, but to be sure that your data is valid is to use both because each method alone has drawbacks For example if a user decides to disable javascript on the front-end all validation will be useless and the server will start receiving invalid data.
